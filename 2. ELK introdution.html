Elastic Search

Elastic Search es una base de datos utilizada para almacenar registros de toda clase de fuente (windows, C, firewall, etc.). Elastic Search usa el lenguaje ESQL para realizar búsquedas, tambien usa restful apis y Json, lo que significa que puede usar varias aplicaciones para interactuar con su base de datos de una manera programable. ___ #### Logstash Logstash recopila los datos de diversas fuentes, y los REENVÍA a Elastic Search, al reenviar ==normaliza== los datos para que puedan ser trabajados con Elastic Search. Es posible filtrar por ID de eventos u otros criterios, lo que ayudará a reducir el costo de la ingesta de Elastic Search.

PARSING (o análisis de campos): A los elementos de un registro se les asignará una palabra clave que actuará como nombre de campo: Sin embargo, no todos los registros tendrán un parsing listo para funcionar, por lo que es posible crear y configurar los nuestros, para extraer ciertos valores de los registros y asignarlos a un nombre de campo. Esta es una habilidad poderosa si se aprende a hacer.

Existen 2 formas de recopilar datos: Beats y Elastic Agent:

  • Beats: existen 6 tipos de archivos diferentes para Beats, para recopilar la telemetría deseada, se debe instalar el Beat apropiado en el endpoint.

  • Elastic Agent: esta es la segunda forma de recopilar datos de telemetría. Puede recopilar todos los tipos de datos usando un agente unificado por host.

Kibana

Kibana nos permite tener una web GUI para consultar nuestros registros almacenados dentro de nuestra instancia de Elastic Search.

Funciones integradas en Kibana: - Data Exploration: Especialmente la pestaña “Discover”, esta permite utilizar consultas ESQL para consultar los registros. - Machine Learning for anomaly detection - Elastic Maps for Geospatial Information - Kibana Lens: esta es una función integrada tambien conocida como ==Visualizations==, donde se puede arrastrar y soltar elementos para ayudar a construir los paneles.

Todo esto permite que Kibana: - Busque datos rápidamente - Cree informes de visualizaciones - Crear Alertas

Principales beneficios de usar ELK

  1. Proporciona un registro centralizado
  2. Flexibilidad al momento de realizar la ingesta de registros (vía Beats o Agentes)
  3. Crear visualizaciones que proporcionen información importante de un vistazo
  4. Escalabilidad para entornos mas grandes o en crecimiento
  5. Integraciones para ELK disponibles en la comunidad, además muchos SIEM se basan en ELK, por lo que migrar a otros entornos podría ser bastante fácil
